# Software Engineering — Reading Order

## Path: Beginner → Advanced

1. **The Linux Command Line** — William Shotts
   If you're not fluent on the shell yet, start here. Foundation for everything else.

2. **Pro Git** — Chacon/Straub
   Git, properly. Don't skip — you'll use Git daily for the rest of your career.

3. **Software Engineering at Google** — Winters et al.
   How a top engineering org actually operates: code review, testing culture, dependency management, SRE.
   This is what "senior SWE" thinking looks like.

4. **Producing Open Source Software** — Karl Fogel
   How OSS communities work — useful even if you don't run a project (you'll be contributing to many).

5. **Crypto 101** — Laurens Van Houtven
   Practical cryptography for engineers — what to use, what NOT to roll yourself.

## What's missing (worth buying)
- **The Pragmatic Programmer** (Hunt/Thomas) — paid, classic
- **Clean Code / Clean Architecture** (Martin) — paid, debated but influential
- **Refactoring** (Fowler) — paid, the canonical refactoring book
