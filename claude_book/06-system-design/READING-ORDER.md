# System Design — Reading Order

## Current contents
1. **Cloud Design Patterns** — Microsoft
   Catalog of cloud architecture patterns (caching, retry, circuit breaker, CQRS, etc.). Practical reference.

## Important note
System design is **mostly NOT learned from books** — it's learned from:
- **Free GitHub repos** — `donnemartin/system-design-primer`, `ByteByteGoHq/system-design-101`
- **Engineering blog posts** — Netflix, Uber, Stripe, Discord, etc.
- **Mock interviews** — Hello Interview, Tech Dummies on YouTube
- **Real production experience**

So this folder is intentionally light.

## Suggested external resources (free)
- **system-design-primer** — github.com/donnemartin/system-design-primer (the most popular system design resource)
- **System Design 101** — github.com/ByteByteGoHq/system-design-101 (visual explanations)
- **Hello Interview** — hellointerview.com (interview-focused, free reading)
- **High Scalability** blog — highscalability.com

## Worth buying
- **Designing Data-Intensive Applications** — Kleppmann (the canonical book)
- **System Design Interview Vol 1 & 2** — Alex Xu

## Reading order
1. Cloud Design Patterns (this folder)
2. system-design-primer (external)
3. Real architecture case studies on engineering blogs
4. Practice with mock system design interviews
