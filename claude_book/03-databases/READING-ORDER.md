# Databases — Reading Order

## Path: Beginner → Advanced

1. **Database Design 2e** — Adrienne Watt
   Beginner ER modeling, normalization, basic SQL. Foundation.

2. **Architecture of a Database System** — Hellerstein/Stonebraker/Hamilton
   ~100 pages; the best concise overview of how a real RDBMS is built. Read after Watt.

3. **Foundations of Databases** ("Alice book") — Abiteboul/Hull/Vianu
   The theoretical/formal side: relational model, query languages, dependencies, datalog.
   Heavy. Read selectively if you want depth.

4. **Readings in Database Systems** ("Red Book", 5e) — Bailis/Hellerstein/Stonebraker (eds.)
   A curated paper collection — what every grad student in DBs should read. Use as a paper reading list.

5. **Mining of Massive Datasets** — Leskovec/Rajaraman/Ullman
   Modern big-data systems, MapReduce, link analysis, recommendation systems. Bridge to data engineering.

## What's missing (and worth buying)
- **Designing Data-Intensive Applications** — Kleppmann (paid). Industry-standard.
- **Database System Concepts (Korth/Silberschatz)** — paid, classic textbook.

## To go advanced
- **CMU 15-445 (Intro to DB Systems)** — free lectures by Andy Pavlo on YouTube
- **CMU 15-721 (Advanced DBs)** — also free, Pavlo
- Read SIGMOD, VLDB, CIDR papers
