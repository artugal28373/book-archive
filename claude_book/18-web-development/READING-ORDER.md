# Web Development — Reading Order

## Currently sparse

Most web dev knowledge lives in **docs and blogs**, not books. The ecosystem moves too fast for textbooks to keep up.

## What you have
1. **Eloquent JavaScript (4e)** — Marijn Haverbeke
   Best free JS book. Modern, well-written. Start here if you don't know JavaScript.

## What to grab next (FREE)

- **You Don't Know JS, Yet** (1st ed series) — Kyle Simpson — github.com/getify/You-Dont-Know-JS — multiple short books on JS internals (CC-licensed)
- **The Modern JavaScript Tutorial** — javascript.info — most up-to-date
- **MDN Web Docs** — developer.mozilla.org — best web reference, period
- **React Docs** — react.dev — official, excellent
- **HTTP/3 explained** — book.huniverse.io — free
- **High Performance Browser Networking** — Ilya Grigorik — hpbn.co (HTML free)

## What's missing (PAID — worth buying)

- **JavaScript: The Definitive Guide (7e)** — Flanagan ⭐ (the canonical reference)
- **Designing Web APIs** — Jin/Sahni/Shevat
- **CSS: The Definitive Guide (5e)** — Eric Meyer
- **Web Application Hacker's Handbook (2e)** — Stuttard/Pinto (security)

## Path: Beginner → Advanced

### Web fundamentals (week 1-3)
1. HTML/CSS basics — MDN tutorials
2. Eloquent JavaScript chapters 1-7

### Modern JavaScript (week 4-8)
3. Eloquent JavaScript chapters 8-21
4. You Don't Know JS — *Scope & Closures* + *this & Object Prototypes*

### Frontend framework (week 9-16)
5. React (or Vue, Svelte) — start with official docs
6. Build something real — todo app → blog → mini SaaS clone

### Backend (week 17-24)
7. Node.js + Express OR Python FastAPI OR Go (pick one)
8. Build a REST API + connect to a database (Postgres)

### Production concerns (ongoing)
9. Authentication (OAuth2, JWT)
10. Deployment (Docker, Vercel, Railway, Fly.io)
11. Observability (logs, metrics, tracing)
12. Web security (OWASP top 10)

## Reality check
Web dev is the most documentation-driven field in CS. Don't try to learn it from books — learn from official docs + building things. Books are useful only for foundational JS/HTTP knowledge.
